3. 게임 컨셉

핵심 플레이 루프

1. 유저가 스트리머 닉네임/ID 검색
2. 매물이 없으면 → 경매 등록 (큐에 추가)
3. 경매 시작 → 5분간 실시간 입찰 경쟁
4. 최고 입찰자 낙찰 → 매물 보유
   (경매 종료 후 10분 쿨다운 → 다음 경매 자동 시작. 50,000G로 즉시 건너뛰기 가능)

6. 다른 유저가 내 매물에 경매 요청 → 승인/거절
7. 경매에서 더 높은 가격에 판매 → 수익 실현

4. 경매 시스템

4.1 경매 유형 3가지

유형 A — 신규 매물 경매 구현 완료

1. 유저가 스트리머 검색 → 매물 없음 (또는 보유자 없는 매물)
2. 유저가 경매 등록 (시작가 5만G 이상) → 최소 시세 50,000G 즉시 차감
3. 대기열에 추가 (선착순)
4. 순서가 오면 경매 시작 (5분)
5-A. 입찰 있음 → 최고 입찰자 낙찰
5-B. 유찰 → 등록자가 시작가에 자동 낙찰 (반드시 보유자 생김)

유형 B — 보유자 승인 경매 구현 완료

1. 유저B가 유저A의 매물에 경매 요청 (requestAuction)
   → 면역 기간 중 요청 불가 / 이미 대기 중인 요청 있으면 불가
2. 유저A에게 내 보유 매물 패널에 승인/거부 버튼 노출 (24시간 기한)
3-A. 승인 (respondToAuctionRequest approve) → 경매 대기열 추가, 시작가는 유저A가 지정
3-B. 거부 (respondToAuctionRequest reject) → 해당 매물 24시간 면역 설정
3-C. 24시간 미회신 → 강제청산 (보유자 시세 100% 환급 + 무보유 전환)

낙찰 처리:
4-A. 낙찰 (입찰자 존재) → 낙찰자에게 매물 이전, 유저A에게 낙찰가 × 95% 즉시 지급
4-B. 유찰 + 유저B 잔액 충분 → 유저B가 시작가에 자동 낙찰, 유저A에게 시작가 × 95% 지급
4-C. 유찰 + 유저B 잔액 부족 → 유저A 소유 유지 + 해당 매물 24시간 면역 설정

유형 C — 손절 경매 구현 완료

1. 보유자가 본인 매물 경매 등록 (시작가 5만G 이상) — registerAuction type:"selloff"
   → 검색 결과 또는 내 보유 매물 패널의 "손절 등록" 버튼
   → 매물 isLocked: true 설정
2. 5분 경매 진행 (모든 유저 입찰 가능)
3-A. 낙찰 → 낙찰자에게 매물 이전, 보유자에게 낙찰가 × 95% 즉시 지급
3-B. 유찰 → 경매 취소 (보유 유지, 시세 변동 없음, 면역 기간 없음)

4.5 경매 상태

QUEUED     대기열에서 대기 중
ACTIVE     경매 진행 중 (5분 카운트다운)
COMPLETED  낙찰 완료 (실제 입찰 낙찰 또는 유형 B 신청자 자동 낙찰)
COOLDOWN   낙찰/유찰 후 다음 경매 시작 전 10분 대기 (cooldownUntil 타임스탬프 포함)

금고 500,000G 계정에 금고 슬롯 1칸 영구 추가. 슬롯에 보유 매물 1개를 보관하면 강제청산 완전 면제. 손절하면 슬롯이 비워지며 다른 매물에 재사용 가능. 슬롯은 계정에 귀속 (양도 불가). Phase 3
면역 연장권 50,000G 유찰 후 면역 기간 24h → 48h 연장. 대상 매물 보유자만 사용 가능. 구현
거래 보유 한도 +1 300,000G 계정 보유 매물 한도 1칸 영구 확장 (최대 한도 별도 정책). Google 계정 전용. 구현
대기열 우선권 패스 (30일) 300,000G 30일간 등록하는 모든 경매를 대기열 맨 앞으로 이동. 단, 현재 진행 중인 경매는 밀지 않음. 구현
편의 상세 열람 패스 (30일) 300,000G 30일간 모든 매물 상세 페이지 무제한 열람 (건당 50,000G 대신). 경매 히스토리는 별도. 구현
히스토리 패스 (30일) 300,000G 30일간 모든 경매 히스토리 무제한 열람 (건당 50,000G 대신). 구현
코스메틱 닉네임 색상 변경 50,000G 리더보드·채팅에서 닉네임 색상 커스텀 (컬러 팔레트 선택). 30일 유효. Phase 3
특수 칭호 100,000G~ 닉네임 옆에 표시되는 칭호 배지 (예: 🏆 경매왕 · 💎 다이아 보유자 등). 영구. Phase 3
프로필 테두리 80,000G 매물 카드·마이페이지 프로필 이미지에 특수 테두리 적용. 30일 유효. Phase 3

15.3 구매 플로우

1. 상점 페이지(shop.html) 접속
2. 카테고리 탭 선택 → 아이템 카드 필터링
3. 아이템 카드 "구매" 버튼 클릭
4. 구매 확인 모달 표시
   - 아이템명 · 효과 요약 · 차감 G · 구매 후 잔액 미리보기
   - needsTarget 아이템(연장권·면역권)은 대상 매물 선택 드롭다운 표시
     * 강제청산 기간 연장권: 모든 내 보유 매물 표시 (요청 없는 매물 선택 시 예약, 있는 매물 선택 시 즉시 적용)
     * 면역 연장권: 면역 기간 중인 매물만 표시
   - 잔액 부족 시 버튼 비활성화 + 부족 금액 안내
5. 확인 → purchaseShopItem CF 호출 (Firestore 트랜잭션)
   - 잔액 차감 / 패스형은 users.{passField} 만료 timestamp 갱신 / 연장권은 listing 또는 request 직접 갱신
6. 성공 토스트 + 잔액 즉시 갱신
7. 효과 적용 시점: 즉시 적용형(패스, 면역 연장) vs 자동 적용형(강제청산 연장권 예약)

15.4 Firestore 구조

users/{uid}
  balance: number
  ownedLimit: number                    // 보유 한도 (보유 한도+1 구매 시 increment)
  detailViewPassExpiresAt: number       // 상세 열람 패스 만료 timestamp
  historyViewPassExpiresAt: number      // 히스토리 패스 만료 timestamp
  queuePriorityPassExpiresAt: number    // 대기열 우선권 패스 만료 timestamp

listings/{listingId}
  reservedExtensionHours: number        // 강제청산 기간 연장권 사전 구매분 (h 단위 누적)
                                        // requestAuction 호출 시 expiresAt에 자동 반영 후 필드 삭제
  vaultedBy: null | uid                 // (Phase 3) 금고 보관 중인 유저 UID

RTDB auction/current (쿨다운 상태)
  status: "cooldown"
  auctionId: string
  finalPrice: number
  winnerId: string | null
  completedAt: number
  cooldownUntil: number                 // now + 10분 (ms timestamp)

15.5 설계 원칙

원칙내용
No Pay-to-Win아이템이 경매 결과에 직접 영향을 미치지 않음. 우선권·패스는 편의이지 승률 보정 아님.
재사용 제한남용 가능성 있는 아이템은 계정당 사용 빈도 제한 적용.
관리자 제어가격·활성화 여부를 system/shopItems에서 실시간 변경 가능 (재배포 불필요).
Phase 2 선반영상점 (shop.html + purchaseShopItem CF) 은 Phase 2 예정이었으나 MVP에 선반영 구현 완료. 금고(Phase 3)는 미구현.

16. 매물 시세 랭킹 페이지 Phase 2

모든 매물의 현재 시세를 순위별로 보여주는 공개 페이지. 로그인 없이 열람 가능.

16.1 페이지 구성

영역설명
헤더공통 헤더 (로그인 상태에 따라 잔액·마이페이지 표시)
랭킹 탭시세 순위 / 거래 횟수 순위 / 역대 최고가 순위 — 탭 전환으로 정렬 기준 변경
랭킹 테이블순위 · 매물(닉네임 + soopId) · 현재 시세 · 보유자 · 마지막 거래가 · 거래 횟수 표시
매물 링크각 행 클릭 → listing.html?id={listingId} (상세 페이지)
새로고침수동 새로고침 버튼 (실시간 구독 대신 단순 조회)

16.2 랭킹 기준

탭정렬 기준Firestore 쿼리표시 항목
시세 순위currentPrice 내림차순orderBy("currentPrice","desc").limit(50)현재 시세 강조
거래 횟수 순위tradeCount 내림차순orderBy("tradeCount","desc").limit(50)총 거래 횟수 강조
역대 최고가 순위highestPrice 내림차순orderBy("highestPrice","desc").limit(50)역대 최고가 강조

16.3 랭킹 테이블 컬럼

#컬럼설명
1순위1위부터 번호 표시. 1~3위는 🥇🥈🥉 아이콘
2매물프로필 이미지 썸네일 + 닉네임 + soopId
3현재 시세listings.currentPrice (G 단위)
4보유자보유자 닉네임 (없으면 "미보유")
5마지막 거래가listings.currentPrice 갱신 시점의 낙찰가
6거래 횟수listings.tradeCount

16.4 표시 정책

항목정책
모자이크 매물isMosaiced: true인 매물은 닉네임·soopId 블러 처리 (상세 링크는 유지)
차단 매물로그인 유저의 blockedListingIds에 포함된 매물은 목록에서 제외
미보유 매물ownerId: null인 매물도 포함 (시세 기록 있는 경우)
신규 매물한 번도 거래 없는 매물(currentPrice: 0 또는 없음)은 랭킹에서 제외
페이지 크기최대 50위까지 표시 (페이지네이션 없음 — MVP 단순화)

16.5 데이터 조회 방식

항목내용
조회 방식페이지 로드 시 1회 Firestore 쿼리 (실시간 구독 없음)
캐싱브라우저 로컬 캐시 5분 (탭 전환 시 재조회 없이 메모리 데이터 사용)
접근 제어Firestore 보안 규칙: listings 컬렉션 read 공개 (기존 규칙 활용)
Cloud Function별도 CF 없음 — 클라이언트에서 직접 Firestore 쿼리
