## 1. 프로젝트 개요

### 기본 정보

| 항목 | 내용 |
|---|---|
| **프로젝트명** | StreamAuction |
| **한글 표기** | 스트림옥션 |
| **부제** | Trade the Streamers You Love |

### 한 줄 소개

> 스트리머 닉네임을 두고 실시간으로 경쟁 입찰하는 경매 게임. 한 번에 하나의 경매에 모든 관심이 집중된다.

### v2 → v3 핵심 변경

| 항목 | v2 (이전) | v3 (현재) |
|---|---|---|
| 거래 방식 | 1:1 제안 → 승인 | **실시간 경매 (5분)** |
| 동시 거래 | 제한 없음 | **서버 전체 1개 경매만** |
| 손절 | 시스템 즉시 매입 (80%) | **경매로 판매** |
| 시세 결정 | 마지막 거래가 | **낙찰가** |
| 수수료 | 10% (판매자) | **5% (판매자)** |
| 강제청산 | 시세 환급 | **무보유 처리 + 시세 100% 환급** |

## 3. 게임 컨셉

### 핵심 플레이 루프 (v3)

```
1. 유저가 스트리머 닉네임/ID 검색
2. 매물이 없으면 → 경매 등록 (큐에 추가)
3. 경매 시작 → 5분간 실시간 입찰 경쟁
4. 최고 입찰자 낙찰 → 매물 보유
5. 보유 매물의 시세 변동 관찰
6. 다른 유저가 내 매물에 경매 요청 → 승인/거절
7. 경매에서 더 높은 가격에 판매 → 수익 실현
```

### 차별화 포인트: "하나의 경매"

- **서버 전체에서 동시에 1개 경매만 진행**
- 모든 유저의 관심이 하나에 집중 → 실시간 경쟁 극대화
- 스트리밍 시 시청자와 함께 참여 가능
- 경매 대기열로 순서 관리 (선착순, 최대 5개)

### 게임 화폐

- **단위**: G (Gold)
- **최저 입찰가**: 50,000G
- **최소 입찰 단위**: 10,000G
- **표기 원칙**: 절대 "원/달러" 등 실제 통화 단위 사용 금지 (사행성 회피)

## 4. 경매 시스템

### 4.1 경매 유형 3가지

#### 유형 A: 신규 매물 경매

아무도 보유하지 않은 새 스트리머 매물을 경매에 올리는 경우.

```
[흐름]
1. 유저가 스트리머 검색 → 매물 없음
2. 유저가 경매 등록 (시작가 5만G 이상, 등록 수수료 없음)
3. 대기열에 추가 (선착순)
4. 순서가 오면 경매 시작 (5분)
5-A. 입찰 있음 → 최고 입찰자 낙찰
5-B. 유찰 (입찰 0건) → 등록자가 시작가(최저 시세)에 자동 낙찰
```

**유찰 처리**: 등록자가 최저 시세에 자동 입찰 = 등록자가 낙찰됨.
이 설계로 "경매 등록 = 반드시 누군가 보유하게 됨"이 보장됨.

#### 유형 B: 보유자 승인 경매

다른 유저가 보유된 매물에 경매를 요청하는 경우.

```
[흐름]
1. 유저B가 유저A의 매물에 경매 등록 요청
2. 유저A에게 알림
3-A. 유저A가 승인 → 경매 대기열에 추가 → 시작가 5만G
3-B. 유저A가 거절 → 요청 취소 (24시간 면역)
3-C. 유저A가 24시간 미회신 → 강제청산 (§8 참고)
```

**남용 방지:**
- 거절 시 해당 매물 24시간 면역 (같은 유저 아닌, 모든 유저에게)
- 같은 유저가 같은 매물에 3일 내 재요청 불가

**경매 진행:**
- 시작가: 5만G (모든 경매 동일)
- 보유자(유저A)는 자기 매물 입찰 불가
- 낙찰 시: 낙찰가에서 수수료 5% 차감 후 유저A에게 지급

#### 유형 C: 손절 경매

보유자 본인이 매물을 경매에 올려 판매하는 경우.

```
[흐름]
1. 보유자가 본인 매물을 경매 등록 (시작가 5만G)
2. 대기열에 추가
3. 5분 경매 진행
4-A. 입찰 있음 → 최고 입찰자 낙찰, 보유자에게 낙찰가 - 수수료 5% 지급
4-B. 유찰 → 경매 취소 (보유 유지, 매물 상태 변화 없음)
```

**보유자 자기 입찰 불가** (가격 끌어올리기 어뷰징 방지)

### 4.2 경매 진행 규칙

| 항목 | 값 |
|---|---|
| 경매 시간 | **5분** |
| 최저 시작가 | 50,000G |
| 최소 입찰 단위 | 10,000G (현재 최고가 + 1만G 이상) |
| 동시 경매 | **서버 전체 1개만** |
| 대기열 | 선착순, **최대 5개** |
| 스나이핑 방지 | 마지막 **15초** 내 새 입찰 → **+15초 연장** |
| 연장 횟수 | **무제한** |

### 4.3 입찰 규칙

| 항목 | 정책 |
|---|---|
| 입찰금 처리 | **에스크로 (즉시 잔액 차감)** |
| 입찰 취소 | **불가** |
| 경매 취소 | **불가** (등록 후 취소 안 됨) |
| 자기 매물 입찰 | **불가** |
| 익명 유저 입찰 | **가능** (보유 한도 1 적용) |
| 익명 유저 경매 등록 | **가능** |
| 더 높은 입찰 시 | 이전 최고 입찰자에게 **즉시 환불** |

**에스크로 동작:**
```
유저A 잔액 500,000G → 100,000G 입찰 → 잔액 400,000G
유저B가 150,000G 입찰 → 유저A에게 100,000G 환불 → 유저A 잔액 500,000G
유저B 낙찰 → 150,000G 확정 차감
```

### 4.4 낙찰 처리

```
[신규 매물 경매]
- 낙찰자 → 매물 보유자 등록
- 낙찰가 → 시스템 회수 (판매자 없으므로 수수료 없음)
- 낙찰가 = 새 시세

[보유자 승인 경매 / 손절 경매]
- 낙찰자 → 새 보유자
- 기존 보유자 → 낙찰가의 95% 수령 (수수료 5% 차감)
- 낙찰가 = 새 시세
```

### 4.5 경매 상태

```
QUEUED:   대기열에서 대기 중
ACTIVE:   경매 진행 중 (5분 카운트다운)
COMPLETED: 낙찰 완료
FAILED:   유찰 (입찰 0건, 손절 경매에서만)
CANCELLED: 취소 (시스템에 의한 예외 처리)
```

### 4.6 메인 페이지 경매 표시

```
🔴 현재 진행 중인 경매
   매물: 홍길동 (hwt1014)
   현재 최고가: 150,000G (3명 입찰)
   잔여 시간: 2:34

🟡 대기 중인 다음 경매
   매물: 이순신 (lss2025)
   등록자: ab12***
   대기 순서: 1번째

⚪ 경매 없음
   "스트리머를 검색해서 첫 경매를 등록해보세요!"
```

## 7. 매물 등록 정책

### 등록 방식

- 유저가 스트리머 닉네임 + soop ID를 검색
- 매물이 없으면 → **경매 등록** (대기열에 추가)
- 별도의 "매물 등록" 행위 없음 (경매 등록이 곧 매물 생성)

### 입력 검증

| 필드 | 규칙 |
|---|---|
| soop 닉네임 | 한글/외래어/대소문자/특수문자 OK, **이모지 NO** |
| soop ID | **영문 대소문자 + 숫자만** |

검증 정규식:
```javascript
// 닉네임: 이모지 차단
const hasEmoji = /\p{Extended_Pictographic}/u.test(nickname);
// ID: 영숫자만
const isValidId = /^[a-zA-Z0-9]+$/.test(soopId);
```

### 닉네임 + ID 듀얼 키

- soop 정책상 닉네임/ID 중복 금지
- **하나라도 동일하면 같은 매물**
- 검색: ID 우선 매칭 → 없으면 닉네임 매칭
- 검색 입력란 단일화 (자동 판단)

### 프로필 이미지

- soop 프로필 URL 패턴: `https://stimg.sooplive.com/LOGO/{앞2글자}/{id}/{id}.jpg`
- CORS 문제로 검증 어려움 → **프로필 없이도 등록 가능**
- 부적절 프로필은 신고 시스템으로 사후 대응

### 등록 한도

- 없음 (보유 한도로 자동 제한됨)
- 경매 등록 후 유찰 시 등록자가 자동 낙찰 → 보유 한도 소진

## 8. 강제청산 시스템

### 트리거

보유자 매물에 경매 등록 요청 후 **24시간 미회신**.

### 작동 방식

```
0시간:  유저B가 유저A 매물에 경매 등록 요청 → 유저A에게 알림
24시간: 유저A 미회신 → 강제청산 발동
```

### 청산 처리

```
1. 유저A(보유자)에게 현재 시세 100% 환급
2. 매물은 무보유 상태로 전환 (시세 유지)
3. 이후 누구나 경매 등록 가능 (시작가 5만G부터)
```

**시세 100% 환급 이유:**
- 보유자가 의도치 않게 빼앗기는 상황이므로 페널티 없음
- 단, 24시간 이내에 회신(승인/거절)하면 면제

### 남용 방지

| 방어 | 내용 |
|---|---|
| 거절 면역 | 보유자가 거절 시 해당 매물 **24시간 면역** (모든 유저에게) |
| 재요청 제한 | 같은 유저가 같은 매물에 **3일 내 재요청 불가** |

### BM 확장 (Phase 2-3)

| 아이템 | 효과 | 가격 |
|---|---|---|
| 강제청산 방어권 | 회신 기한 24시간 → 72시간 연장 | G 결제 (미정) |
| 금고 사용권 | 매물을 금고에 넣어 강제청산 영구 면제 (매물당 1칸) | G 결제 (미정) |

## 9. 어뷰징 방어 시스템

### 경매 시스템 기반 자동 방어

경매 구조 자체가 많은 어뷰징을 자동 방어함.

| 어뷰징 | v2 방어 | v3 자동 방어 |
|---|---|---|
| 자전거래 | 수수료 10% | 경매는 공개 → 다른 유저가 끼어들 수 있음 |
| 시세 펌핑 | IP 차단 | 자기 매물 입찰 불가 + 공개 경매 |
| 가격 조작 | 수동 탐지 | 경매 기록 전부 공개 (Phase 2) |

### 추가 방어 (유지)

| 항목 | 내용 |
|---|---|
| 거래 수수료 | **5% (판매자 부담)** |
| 동일 디바이스/IP 제한 | 같은 IP에서 경매 등록 + 입찰 동시 차단 |
| 신규 계정 제한 | 가입 후 24시간: 경매 등록 불가 (입찰은 가능) |
| 자기 매물 입찰 금지 | Cloud Function에서 ownerId 체크 |
| 입찰 취소 불가 | 허위 입찰 방지 |
| 에스크로 | 잔액 부족 입찰 불가 (즉시 차감) |

### 행동 패턴 탐지 (Phase 2)

| 항목 | 내용 |
|---|---|
| 순환 거래 탐지 | A→B→C→A 패턴 자동 플래그 |
| 비정상 입찰 탐지 | 시세의 10배 이상 입찰 검토 |
| 봇 탐지 | 분당 N회 이상 입찰 차단 |

## 10. 신고 및 숨기기 시스템

### 개인 차단 (구글 유저만)

- 본인 화면에서만 안 보이도록 처리
- 차단 사유 불필요
- 무제한 차단 가능
- users 문서의 `blockedListingIds` 배열에 저장

### 커뮤니티 신고 (구글 유저만)

**신고 사유 (영문 코드 / 관리자 페이지에선 한글):**

| 영문 코드 | 한글 표시 |
|---|---|
| `inappropriateImage` | 부적절한 프로필 이미지 |
| `profanity` | 비속어/혐오 닉네임 |
| `misinformation` | 잘못된 정보 (실제 스트리머 아님) |
| `other` | 기타 (자유 입력) |

**규칙:**
- 1인당 1매물 1회만 신고 가능
- 누적 **10명** 신고 시 → **자동 모자이크 처리** (기본 아바타로 대체)
- 임계값은 system/config의 `reportThreshold`로 관리 (운영 중 조정 가능)

**신고자 보호:**
- 신고자 정보 비공개 (보안 규칙으로 차단)

**모자이크 처리:**
- 프로필 이미지를 **기본 아바타**로 대체
- 거래 자체는 계속 가능
- 운영자가 Firebase Console에서 검토 후 복원/유지 (관리자 페이지는 Phase 2)

## 부록 B: 의사결정 히스토리

| 결정 사항 | 이유 |
|---|---|
| v3: 1:1 거래 → 실시간 경매 전환 | 스트리밍 콘텐츠 최적화, 시청자 참여 극대화 |
| 서버 전체 1개 경매만 | 트래픽 집중으로 경쟁 극대화 |
| 수수료 10% → 5% | 경매 시스템 자체가 자전거래 방어, 수수료 부담 완화 |
| 강제청산: 시세 100% 환급 | 보유자 보호 (의도치 않은 손실 방지) |
| 손절: 경매로 전환 | 모든 거래를 경매로 통일, 시스템 일관성 |
| 유찰: 등록자 자동 낙찰 | "경매 등록 = 반드시 보유자 생김" 보장 |
| 에스크로 입찰 | 허위 입찰 방지 + 잔액 부족 문제 해소 |
| 경매 히스토리 BM | 방송 콘텐츠로도 가치 있는 자연스러운 수익화 |
| 강제청산 방어권/금고 BM | Pay-to-Survive (P2W 아님), 유저 동기 강함 |
| 익명도 입찰/등록 가능 | 진입 장벽 낮춤, 체험 후 구글 전환 유도 |
| 입찰/경매 취소 불가 | 허위 입찰 + 가격 조작 방지 |
